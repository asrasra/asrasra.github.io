HTML Cheatsheet

A collection of <html> tags and attributes to reference when building websites ;)

The Basics

To start a documnent you need to declare<!DOCTYPE html>
It let's the browser know that this document is using HTML5

Here are the basic tags that your html document needs(or should) have:

  • <!DOCTYPE html>
  • <html lang="en">
    • <head> <!-- Metadata goes here -->
      • <title>The Title of the Page Goes Here</title>
      • <link href="./styles.css" rel="stylesheet" />
    • </head>
    • <body> <!-- Every visible element of the page -->
      • <header>
        • <nav>
        • </nav>
      • </header>
      • <main> <!-- Main content of the page -->
      • </main>
      • <footer>
      • </footer>
    • </body>
  • </html>

Sections or Articles

Articles and sections are for SEO purposes and define the content of the html.

They can be used as sections in articles or articles in sections. The benefit of using one over the other is that articles are usually what is pulled from the page when linking to that section of an article.

If a page has a blog and the sections are tutorials and reviews, then each individual article topic in that section should have the article tag. Or if within an article page you have different sections that focus on different viewpoints or topics you can use sections within articles to define that change.

Should I use a div?

I used to use divs with classes for 90% of the document. Apparently that is not a good practice of using semantic html . Instead we should take advantage of the sections and articles to define the main content of the page. Divs should probably only be used when needed such as making a shape with CSS, etc.

Tables

To create tables in <html> you will need the following tags and attributes

Tag Description
<table> Table wrapper element
<thead> Table headers
<th> Table headers data/columns
<tbody> Table body
<tr> Table rows
<td> Table data/columns
<tfoot> Table footer
Attribute Description
colspan Defines how many columns a <td> element should span.
rowspan Defines how many rows a <td> element should span.

So realistically, when am I going to use tables in html.. Hopefully never.
